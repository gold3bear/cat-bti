# CBTI 题目设计 v1

> Cat Bitch Test Index · 问题文案 + 双轨权重标注  
> 共 15 题：14 道维度题 + 1 道彩蛋题（Q11）  
> 供开发人员实现算法权重写入，配合《CBTI_开发说明文档_v1.md》使用

---

## 一、设计原则

- **表面测猫，暗里测人**：每道题的问题主语是猫，但选项设计带入主人心理
- **幽默不油腻**：选项文案用铲屎官日常语气，让测试者"笑着被说中"
- **投射可信**：每道题的选项要让人觉得"对，我猫就是这样"，而不是猜测题
- **aha铺垫**：选项文案里要埋下让人回头想"等等，这说的是我"的细节

## 三、权重格式说明

每道题每个选项携带两套向量权重：

```
cat[D2, D7, D3, D6]   ← 猫维度，用于类型匹配
human[D2, D7, D3, D6] ← 人维度，用于镜像/代偿判断 & 隐藏款触发
```

数值范围：`-2`（低极极端） → `0`（中性） → `+2`（高极极端）

## 五、分维度得分规则

每个维度从参与该维度的题目累加猫向量得分：

| 维度 | 参与题目 | 满分范围 | H阈值（高极） | L阈值（低极） |
|------|---------|---------|------------|------------|
| D2 情绪表达 | Q2, Q4, Q6, Q15（部分） | -8 ~ +8 | ≥ +3 → E | ≤ -3 → S |
| D7 社交模式 | Q1, Q5, Q10, Q15（部分） | -8 ~ +8 | ≥ +3 → O | ≤ -3 → I |
| D3 控制欲 | Q3, Q7, Q8, Q9 | -8 ~ +8 | ≥ +3 → C | ≤ -3 → F |
| D6 节律 | Q12, Q13, Q14 | -6 ~ +6 | ≥ +2 → R | ≤ -2 → X |

> **注**：部分题目（如 Q8、Q9）的选项同时携带 D3 + D6 双维度权重，权重值参考第四节各题权重表。

得分结果为 H/L/M（M 为中间值，算法上取更接近的极端方向，若完全中立则取 H 作为默认）。

## 六、隐藏款触发逻辑

### ANCE · 猫祖宗（约 0.3%）

```
触发条件：
  Q11 选择 D（easter = 3）
  AND D3控制欲猫向量总分 ≥ +5（极端 C）
```

逻辑说明：把手机完全让给猫 + 控制欲极端高 = 把整个自我都寄存在猫身上的人

---

### ACT · 戏精绿茶（约 5%）

```
触发条件：
  人向量 D2 总分 ≥ +4（高情绪 E）
  AND 人向量 D7 总分 ≤ -4（极端社恐 I）
  （情绪外放但极度内向的强烈反差）
```

逻辑说明：情绪表达丰富但社交极度回避 = 会演、在外包装、但本质独处型，就是"戏精绿茶"的人格组合

---

### RARE · 爵士好猫（约 3%）

```
触发条件：
  猫向量四个维度得分均在 -2 ~ +2 之间（无明显极端值）
  AND 四维度总均值 ≥ 0（整体偏中高均衡）
```

逻辑说明：四维均衡无极端、整体偏正 = 选了一只"说不清类型"的猫，是有不想被归类的那一面的主人

## 七、隐藏款优先级

算法执行顺序：

```
1. 收集答题数据 → 计算猫向量 + 人向量
2. 先判断 ANCE 触发（Q11 + D3极端）
   ├── 满足 → 返回 ANCE，跳过后续
   └── 不满足 → 继续
3. 判断 ACT 触发（人向量 D2/D7 极端反差）
   ├── 满足 → 返回 ACT，跳过后续
   └── 不满足 → 继续
4. 判断 RARE 触发（四维均衡）
   ├── 满足 → 返回 RARE，跳过后续
   └── 不满足 → 进入主型匹配
5. 主型欧氏距离匹配 → 返回 16 主型中最近的结果
6. 计算 match_score = 100 - (最小距离 / 最大距离 × 100)
```
