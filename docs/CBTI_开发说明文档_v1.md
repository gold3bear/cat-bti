## 一、产品核心理念

### 1.1 一句话定义

> **你以为在测猫，测出来的是你自己。**

CBTI 的核心机制是**反转**：用户全程相信自己在描述猫的行为，答题结束后结果页第一屏给出猫的人格类型，第二屏才揭示「这实际上说明你是什么样的人」。反转要做到：**先惊喜，后 make sense**。

### 1.2 区别于 MBTI / SBTI 的核心差异

| 维度 | MBTI | SBTI | CBTI |
|------|------|------|------|
| 测的对象 | 自己 | 自己 | 表面测猫，实际测主人 |
| 命名逻辑 | 维度首字母拼词 | 先找梗词再配维度 | 同 SBTI，先找中英文梗词 |
| 结果数量 | 16型 | 25型 | 16主型 + 3隐藏款 |
| 传播钩子 | 身份认同 | 幽默自嘲 | 反转惊喜 + 分享给铲屎官朋友 |
| 核心情绪 | 「这说的是我」 | 「哈哈这也太准了」 | 「我去，测的是猫但说的是我？」 |

### 1.3 三大心理学原理

**① 巴纳姆效应（Barnum Effect）**
每个维度的两极描述都能让用户感到「说的是我」，不存在明显「坏的那端」。选词时优先保证两端都是合理的人格状态。

**② 投射心理学（Projection Psychology）**
用户回答猫的行为时，无意识地将自身心理状态投射到描述中。维度越心理化（而非纯行为化），投射越强。这是为何最终选择「情绪表达/社交模式/控制欲/节律」而非「活跃度」等纯行为维度。

**③ 互补镜像算法（Complementary Mirror）**
结果分为两种类型：
- **同向镜像型**：你养了一只和你一样的猫（镜子逻辑）
- **互补代偿型**：你的猫有你没有的东西（你缺什么，猫就补什么）

两种类型交替出现，保证不同用户都能找到共鸣。

## 二、四维度设计

### 2.1 维度说明

CBTI 使用 **4 个二元维度**，每维度各取一个字母，组成 4 字母类型代码（如 MONK、FIRE）。

| 维度编号 | 维度名称 | 低极（L） | 高极（H） | 低极字母 | 高极字母 | 选择理由 |
|---------|---------|---------|---------|---------|---------|---------|
| D2 | 情绪表达 | 压抑 / 冷漠 | 外放 / 热情 | S (Stoic) | E (Expressive) | 投射强度最高，用户描述猫的「冷漠/黏人」本质上是描述自己的情感状态 |
| D7 | 社交模式 | 回避 / 独处 | 寻求 / 外向 | I (Introverted) | O (Open) | MBTI E/I 轴的直接对应，巴纳姆效应最强的维度 |
| D3 | 控制欲 | 独立 / 自由 | 掌控 / 黏人 | F (Free) | C (Control) | 猫「不理人」是铲屎官最普遍的共同经历，投射到主人控制欲极具穿透力 |
| D6 | 节律 | 混沌 / 随性 | 规律 / 精准 | X (Xchaos) | R (Regular) | MBTI J/P 轴对应，「猫的混沌作息 vs 主人的强迫规律」是高共鸣场景 |

### 2.2 编码规则

```
类型代码 = [D2字母][D7字母][D3字母][D6字母]

示例：
E + O + C + R = EOCR（高情绪 + 高社交 + 高控制 + 高规律）
S + I + F + X = SIFX（低情绪 + 低社交 + 低控制 + 低节律）
```

> **注意**：CBTI 的 4 字母代码**不直接对应类型英文词**。  
> 英文词（MONK、FIRE 等）是独立命名的梗词，参考 SBTI 命名逻辑：先找最贴切的英文词，再与维度向量匹配。  
> 英文词与 4 字母代码是**两套独立标识**，前者面向用户展示，后者面向算法计算。

## 三、类型结构

### 3.1 总体分布

```
19 种结果 = 16 主型 + 3 隐藏款

主型：正常答题流程中必然命中其中一种
隐藏款：需满足特定触发条件才能出现，不在常规路径中
```

### 3.2 隐藏款触发条件

| 代码 | 中文名 | 触发条件 | 触发概率 |
|------|--------|---------|---------|
| ANCE | 猫祖宗 | Q11 彩蛋题选择特定答案 + D3控制欲维度得分极端高 | 约 0.3% |
| ACT | 戏精绿茶 | D2情绪表达与D7社交模式出现极端反差（一高一低且均为极端值） | 约 5% |
| RARE | 爵士好猫 | 四个维度得分均匀分布，无明显极端，且总体趋向中高均衡 | 约 3% |

> **实现建议**：隐藏款触发逻辑优先级高于主型匹配。先检测是否满足隐藏款触发条件，满足则跳过主型距离匹配，直接输出对应隐藏款。

## 四、结果输出字段格式

每次测试完成后，系统需返回一个**结果对象**，包含以下字段。

### 4.1 字段总览

```json
{
  "type_code": "EOCX",
  "en_word": "FIRE",
  "zh_name": "显眼包",
  "is_hidden": false,
  "rarity_label": "自带扩音器款",
  "rarity_pct": 14,
  "mirror_type": "同向镜像",
  "cat_behavior": "...",
  "owner_projection": "...",
  "aha_moment": "...",
  "gold_quote": "...",
  "trigger_condition": null,
  "dimension_scores": {
    "D2_emotion": "H",
    "D7_social": "O",
    "D3_control": "C",
    "D6_rhythm": "X"
  },
  "match_score": 94.2
}
```

### 4.2 字段逐一说明

---

#### `type_code`
- **类型**：String
- **示例**：`"EOCX"`
- **说明**：由四维度字母拼接而成的算法标识符。是系统内部唯一主键，用于关联文案、图片、分享卡片等所有资源。**开发时所有资源映射均以此字段为准**，不要用 `en_word`。
- **重要性**：⭐⭐⭐⭐⭐ 核心标识，不可缺失

---

#### `en_word`
- **类型**：String
- **示例**：`"FIRE"`
- **说明**：面向用户展示的英文梗词，出现在结果卡片顶部大字和分享图中。与 `type_code` 是两套独立标识，`en_word` 不参与算法计算，仅用于展示。
- **重要性**：⭐⭐⭐⭐⭐ 用户记忆锚点，传播核心

---

#### `zh_name`
- **类型**：String
- **示例**：`"显眼包"`
- **说明**：中文类型名称，紧跟在 `en_word` 下方展示。与英文词共同构成「FIRE · 显眼包」的双语标题格式。
- **重要性**：⭐⭐⭐⭐⭐ 用户识别核心

---

#### `is_hidden`
- **类型**：Boolean
- **示例**：`false` / `true`
- **说明**：标识该结果是否为隐藏款。隐藏款在结果页需要额外展示「隐藏款触发」的特殊视觉效果（如金色边框、稀有感动画）。
- **重要性**：⭐⭐⭐⭐ 影响结果页 UI 分支逻辑

---

#### `rarity_label`
- **类型**：String
- **示例**：`"自带扩音器款"`
- **说明**：稀有度的文案标签，显示在结果卡片稀有度区域。格式为「XX款」，强调用户身份感，是巴纳姆效应的重要触点——每个标签都让用户感到「这就是我的专属款」。
- **重要性**：⭐⭐⭐⭐ 影响用户对结果的认同感

---

#### `rarity_pct`
- **类型**：Integer（百分比数值，不含 % 符号）
- **示例**：`14`
- **说明**：稀有度百分比，配合 `rarity_label` 展示为「自带扩音器款 · 14%」。数值不追求精确，服务于「稀有感」体验——越低越珍贵，促进截图分享。隐藏款数值建议 0.3%~5%。
- **重要性**：⭐⭐⭐⭐ 分享传播的重要驱动因子

---

#### `mirror_type`
- **类型**：String（枚举）
- **枚举值**：`"同向镜像"` / `"互补代偿"`
- **说明**：结果的心理学逻辑类型。**此字段不对用户展示**，供内部文案校验和后续数据分析使用。同向镜像：猫是主人的镜子；互补代偿：猫补充了主人缺失的部分。
- **重要性**：⭐⭐⭐ 内部字段，用于文案一致性校验

---

#### `cat_behavior`
- **类型**：String（富文本，支持换行）
- **示例**：`"你的猫进门第一件事：找到全场最重要的东西把它打翻……"`
- **说明**：占结果文案约 30% 的猫行为描述段落。**语气是客观描述猫的行为**，让用户先进入「对，这就是我猫」的认同状态，为后续投射和反转铺垫。此段不应出现任何对主人的心理分析。
- **重要性**：⭐⭐⭐⭐⭐ 反转机制的前置铺垫，不可省略

---

#### `owner_projection`
- **类型**：String（富文本，支持换行）
- **示例**：`"FIRE猫和主人是频率相同的生物。你不一定在外面出风头……"`
- **说明**：占结果文案约 70% 的核心内容，实现从猫行为到主人心理的反转解读。这是整个测试的灵魂所在：**用户在这里经历「aha时刻」**，从「我在描述猫」切换到「这说的是我自己」。文案采用第二人称「你」，加强代入感。
- **重要性**：⭐⭐⭐⭐⭐ 产品核心体验，传播驱动力来源

---

#### `aha_moment`
- **类型**：String（单句，通常加粗展示）
- **示例**：`"我来了，你们都应该知道。"`
- **说明**：整段文案中最核心的一句反转句，通常加粗或换色展示，是用户截图分享时最常截取的部分。句式参考：「你不是X，你压根是Y」/ 「它有X，因为你需要X」/ 「说的是猫，还是你」。
- **重要性**：⭐⭐⭐⭐⭐ 分享传播的核心句，决定用户是否发朋友圈

---

#### `gold_quote`
- **类型**：String（带引号，单句）
- **示例**：`"「低调留给不需要证明自己的人。你和你的猫，都不是那种人。」"`
- **说明**：结尾金句，单独成行，带书名号或引号，视觉上独立展示。功能类似「星座金句」，是用户截图分享时的第二高频截取内容。金句要有通用性（巴纳姆），也要有锋芒（让人想发朋友圈挑衅朋友「你也是这个」）。
- **重要性**：⭐⭐⭐⭐⭐ 分享传播的视觉锚点

---

#### `trigger_condition`
- **类型**：String | null
- **示例**：`"Q11彩蛋题选择特定答案 + D3控制欲维度极端高"` / `null`
- **说明**：仅隐藏款有值，主型为 `null`。描述触发该隐藏款的条件，供算法层判断使用。隐藏款触发优先级高于主型匹配，算法应先完成隐藏款判断再进行主型向量距离计算。
- **重要性**：⭐⭐⭐⭐ 隐藏款机制实现的关键字段

---

#### `dimension_scores`
- **类型**：Object
- **示例**：
  ```json
  {
    "D2_emotion": "H",
    "D7_social": "O",
    "D3_control": "C",
    "D6_rhythm": "X"
  }
  ```
- **说明**：用户在四个维度上的最终得分分级（H=高极，L=低极，M=中间，字母为对应高极字母）。用于结果页「维度雷达图」或「你的猫性格分析」可视化模块的数据源。也可用于后台统计各维度分布。
- **重要性**：⭐⭐⭐ 可视化模块数据源，可选展示

---

#### `match_score`
- **类型**：Float（保留一位小数）
- **示例**：`94.2`
- **说明**：用户向量与该类型原型向量的匹配度，以百分比形式展示（参考 SBTI「匹配度 XX%」）。计算方式：`100 - (欧氏距离 / 最大距离 × 100)`。此字段直接在结果页展示，强化「科学感」，同时满足巴纳姆效应（高匹配度让用户感到被精准描述）。建议展示范围 75%~99%。
- **重要性**：⭐⭐⭐⭐ 增强结果可信度和分享欲

## 五、算法逻辑简述

### 5.1 双轨向量系统

CBTI 使用**双轨并行向量**计算，每道题的每个选项同时携带两套权重：

```
答题 → 每个选项写入：
  [猫维度权重 D2c / D7c / D3c / D6c]  ← 用于算法匹配类型
  [人维度权重 D2h / D7h / D3h / D6h]  ← 用于结果文案的反转解读深度
```

猫向量用于距离匹配，人向量用于判断文案中「同向镜像」还是「互补代偿」逻辑，以及隐藏款触发判断。

### 5.2 类型匹配流程

```
1. 收集用户答案 → 计算用户猫向量
2. 检查隐藏款触发条件（优先级最高）
   ├── 满足 → 直接返回对应隐藏款结果
   └── 不满足 → 进入主型匹配
3. 计算用户猫向量与 16 个主型原型向量的欧氏距离
4. 取最小距离对应的主型 → 返回主型结果
5. match_score = 100 - (最小距离 / 最大距离 × 100)
```

### 5.3 题目结构参考

- **总题数**：15 题（14 维度题 + 1 彩蛋题 Q11）
- **每题选项**：4 个（每个选项携带猫/人双权重）
- **维度分布**：每维度约 3~4 题，彩蛋题 1 题
- **Q11 彩蛋题**：特殊题，答案影响 ANCE 隐藏款触发逻辑

## 六、结果页 UI 展示建议

### 6.1 两屏揭示结构（核心体验）

```
第一屏（猫型揭示）
├── 稀有度标签 + 百分比（rarity_label + rarity_pct）
├── 英文大字（en_word）
├── 中文名（zh_name）
├── 匹配度（match_score）
└── [点击查看完整解读] 按钮

↓ 用户点击或下滑 ↓

第二屏（主人反转揭示）
├── 猫行为描述（cat_behavior）
├── 主人投射（owner_projection）
├── Aha 核心句加粗展示（aha_moment）
└── 结尾金句（gold_quote）
```

> 建议第一屏和第二屏之间加入 1.5~2 秒的**悬念延迟动画**，强化反转惊喜感。

### 6.2 分享卡片字段优先级

分享图中应包含（按重要性排序）：

1. `en_word` + `zh_name`（最大字号）
2. `aha_moment`（第二字号，加粗）
3. `gold_quote`（引号格式）
4. `rarity_label` + `rarity_pct`（角标或徽章形式）
5. CBTI 品牌标识 + 测试链接
